## 브라우저란

브라우저는 <strong>"웹페이지를 표시하고 사용자가 웹페이지와 상호작용할 수 있게 해주는 소프트웨어"</strong>입니다.

+ **동작 방식:** 사용자가 URL을 입력하면 브라우저는 해당 서버에 요청을 보내고, 응답으로 받은 HTML, CSS, JavaScript 리소스를 해석하여 화면에 출력합니다.

+ **상호작용:** 링크 클릭, 폼 입력, 데이터 제출 등 사용자의 동작을 서버로 전달하는 인터페이스 역할을 수행합니다.

## 브라우저 구성 요소

+ **사용자 인터페이스 (User Interface):** 주소 표시줄, 이전/다음 버튼, 북마크 등 페이지 뷰를 제외한 모든 영역.

+ **JavaScript 해석기 (JS Engine):** 웹페이지의 동적 기능을 구현하는 자바스크립트 코드를 해석하고 실행함 (예: Chrome의 V8).

+ **네트워킹 (Networking):** HTTP 요청을 통해 서버와 통신하고 리소스를 받아오는 역할.

+ **데이터 저장소 (Data Storage):** 쿠키, 로컬 스토리지, 세션 스토리지 등 브라우저 내부에 데이터를 저장하는 공간.

+ **렌더링 엔진 (Rendering Engine):** HTML과 CSS를 해석하여 화면에 시각적으로 그리는 핵심 요소.

## 렌더링 과정 (Rendering Pipeline)

웹 페이지의 내용이 사용자의 화면에 표시되는 일련의 과정을 **렌더링**이라고 합니다.

1. **파싱 (Parsing):** HTML 문서를 읽어 문서의 구조를 분석하고 **DOM(Document Object Model) 트리**를 생성합니다.

2. **스타일링 (CSS Parsing):** CSS 파일을 해석하여 각 요소에 적용될 스타일 정보를 담은 **CSSOM(CSS Object Model) 트리**를 생성합니다.

3. **레이아웃 (Layout):** DOM과 CSSOM을 결합하여 실제 화면에 보일 **Render Tree**를 만들고, 각 요소의 위치와 크기를 계산합니다.

4. **페인팅 (Painting):** 계산된 레이아웃 정보를 바탕으로 화면에 실제 픽셀(색상, 이미지 등)을 그려줍니다.

## HTML5 란?

**HTML5**는 웹 개발에서 사용되는 최신 마크업 표준 언어입니다.<br>
Hyper Text Markup Language의 5번째 버전으로, 별도의 플러그인 없이 비디오/오디오 재생이 가능하며 의미 있는 태그(Semantic Tags)를 제공합니다.

### 마크업 언어 (Markup Language)의 특징

+ **구조화:** 문서의 콘텐츠(내용)가 어디에 위치하고 어떤 역할을 하는지 정의합니다.

+ **분리:** 콘텐츠와 서식(디자인)을 분리하여, 내용 수정 없이도 레이아웃이나 스타일을 쉽게 변경할 수 있습니다.

+ **예시:** HTML, XML, Markdown 등이 대표적입니다.

## TIP) 인터페이스 전송 데이터

서버와 클라이언트가 데이터를 주고받을 때 사용하는 표준 포맷들입니다.

+ **JSON (JavaScript Object Notation):** 자바스크립트 객체 문법을 따르는 문자 기반 데이터 포맷. 현재 가장 많이 사용됨.

+ **XML (eXtensible Markup Language):** 태그를 이용해 데이터를 구조화한 마크업 언어.

+ **CSV (Comma Separated Values):** 쉼표(,)로 데이터를 구분한 텍스트 파일. 엑셀과 호환성이 높음.

+ **YAML:** 사람이 읽기 쉬운 데이터 직렬화 양식. 주로 서버 설정 파일에 사용됨.
