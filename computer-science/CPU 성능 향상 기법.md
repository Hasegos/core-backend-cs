## 기본 성능 지표 (클럭, 코어, 스레드)

+ **클럭 (Clock):** CPU의 속도 단위(Hz). 1초에 클럭이 몇 번 반복되는지를 의미합니다.

+ **코어 (Core):** 명령을 해석하고 실행하는 실질적인 "부품". 다중 코어(Multi-core)일수록 병렬 처리에 유리합니다.

+ **스레드 (Thread):**
    + **하드웨어적 스레드:** 하나의 코어가 동시에 처리하는 논리적인 작업 단위 (**논리 프로세서**).

    + **소프트웨어적 스레드:** 하나의 프로그램 내에서 독립적으로 실행되는 작업 흐름.

## 명령어 파이프라이닝 (Pipelining)

명령어 처리 단계를 나누어 마치 공장의 컨베이어 벨트처럼 여러 명령어를 **동시에 실행**하는 기법입니다.

### 파이프라인 위험 (Pipeline Hazard)

성능 향상을 방해하는 세 가지 주요 원인입니다.

+ **데이터 위험 (Data Hazard):** 이전 명령어의 결과값이 나와야 다음 명령어를 실행할 수 있는 의존성 문제.

   + *예: R1 = R2 + R3 수행 후, R4 = R1 + R5 수행 시 발생*

+ **제어 위험 (Control Hazard):** `JUMP`나 `IF`문 등으로 인해 프로그램 카운터(PC)가 갑작스럽게 변할 때 발생.

+ **구조적 위험 (Structural Hazard):** 서로 다른 명령어가 동시에 같은 하드웨어 자원(ALU, 레지스터 등)을 사용하려 할 때 발생.

### 슈퍼스칼라 (Superscalar)

CPU 내부에 여러 개의 파이프라인을 두어 한 클럭에 여러 명령어를 동시에 실행하는 구조입니다. 현대 CPU의 핵심 기술입니다.

## 비순차적 명령어 처리 (OoOE)

<strong>OOOE (Out-of-Order Execution)</strong>는 명령어의 순서를 데이터 의존성이 없는 범위 내에서 **합법적으로 새치기**하여 파이프라인 정체(Stall)를 막는 기법입니다.

+ **원리:** 순차적으로 기다리면 노는 시간이 생기므로, 먼저 처리 가능한 뒤쪽 명령어를 미리 실행합니다.

+ **효과:** 파이프라인의 데이터 위험 등으로 인한 대기 시간을 최소화합니다.

## ISA (명령어 집합 구조): CISC vs RISC

ISA는 CPU가 이해할 수 있는 명령어들의 약속(설계도)입니다.

### CISC (Complex Instruction Set Computer)

+ **특징:** 복잡하고 다양한 기능을 가진 가변 길이 명령어 집합.

+ **장점:** 컴파일된 명령어 수가 적어 메모리 절약 가능.

+ **단점:** 명령어마다 실행 시간이 달라 파이프라이닝이 어렵고 효율이 낮음 (Intel x86 등).

### RISC (Reduced Instruction Set Computer)

+ **특징:** 단순하고 규격화된 고정 길이 명령어 집합.

+ **장점:** 1클럭 내외로 실행되어 파이프라이닝에 최적화됨. 성능이 좋음.

+ **단점:** 컴파일된 명령어 수가 CISC보다 많음 (ARM, Apple Silicon 등).

### CISC vs RISC 비교

| 구분 | CISC | RISC |
| :--- | :--- | :--- |
| **명령어 종류** | 복잡하고 다양함 | 단순하고 적음 |
| **명령어 길이** | **가변 길이** | **고정 길이** |
| **수행 시간** | 여러 클럭 소요 | 1클럭 내외 |
| **파이프라이닝** | 어려움 | **매우 유리함** |
| **사용처** | PC, 서버 (Intel) | 모바일, 임베디드 (ARM) |
