## 캐시 메모리의 역할

CPU가 주기억장치에서 데이터를 읽어올 때, 자주 사용하는 데이터를 캐시에 저장해 두었다가 다시 사용할 때 빠르게 가져옴으로써 전체적인 시스템 성능을 향상시킵니다.

+ **장점:** 데이터 접근 속도가 매우 빠름

+ **단점:** 용량이 적고 가격이 매우 비쌈

## 캐시 메모리 계층 구조

CPU 내부와 주변에는 속도와 크기에 따라 여러 단계의 캐시가 존재합니다.

+ **L1 Cache:** CPU 내부에 존재하며 가장 빠름. (Instruction/Data 캐시로 분리되기도 함)

+ **L2 Cache:** CPU와 RAM 사이에 존재하며 L1보다 크지만 조금 더 느림.

+ **L3 Cache:** 주로 메인보드나 CPU 패키지에 공유 캐시 형태로 존재함.

+ **디스크 캐시:** RAM과 하드디스크 사이의 병목 현상을 줄이기 위해 사용됨.

## 작동 원리 (참조 지역성)

캐시는 효율을 극대화하기 위해 **참조 지역성(Locality of Reference)** 원리를 이용합니다.

+ **시간 지역성 (Temporal Locality):** 최근에 참조된 주소는 곧 다시 참조될 가능성이 높음 (예: 반복문의 조건 변수).

+ **공간 지역성 (Spatial Locality):** 참조된 주소와 인접한 주소의 데이터가 곧 참조될 가능성이 높음 (예: 배열 연속 접근).

> **결과:** CPU가 요청한 데이터가 캐시에 있으면 **Cache Hit**, 없어서 메모리에서 가져오면 **Cache Miss**라고 합니다.

## 캐시 미스 (Cache Miss) 3가지

+ **Cold Miss:** 해당 메모리 주소를 처음 불러서 발생하는 미스.

+ **Conflict Miss:** 서로 다른 데이터가 동일한 캐시 주소에 할당되어 발생하는 충돌 미스 (Direct Mapped 방식에서 빈번).

   + 비유: **우체통 하나에 여러 집의 우편물을 넣어야 해서 기존 우편물이 밀려나는 상황**

+ **Capacity Miss:** 캐시 메모리의 전체 공간이 부족하여 발생하는 미스.

## 캐시 매핑 방식

메모리의 데이터를 캐시의 어디에 저장할지 결정하는 방식입니다.

### ① Direct Mapped Cache

가장 기본적인 구조로, 메모리 주소와 캐시 주소가 **1:N**으로 매핑됩니다.

+ **장점:** 구현이 간단하고 접근 속도가 빠름.

+ **단점:** 동일 인덱스에 매핑되는 주소가 많아 **Conflict Miss**가 잦음.

### ② Fully Associative Cache

비어있는 캐시 공간 어디에나 데이터를 저장할 수 있는 방식입니다.

+ **장점:** 저장이 자유롭고 충돌 미스가 적음.

+ **단점:** 특정 데이터를 찾을 때 모든 슬롯을 검색해야 하므로 검색 속도가 느리고 비용이 비쌈 (CAM 메모리 필요).

### ③ Set Associative Cache

위의 두 방식을 혼합한 방식으로, 특정 행(Set)을 지정하고 그 안에서 비어있는 열(Way)에 저장합니다.

+ 현대 CPU에서 가장 많이 사용하는 중간 형태의 방식입니다.
