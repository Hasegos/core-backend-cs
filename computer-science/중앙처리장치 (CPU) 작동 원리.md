## CPU의 3대 구성 요소

### ① 연산장치 (ALU - Arithmetic Logic Unit)

+ 산술 연산(+, -)과 논리 연산(AND, OR)을 수행합니다.

+ 레지스터에서 데이터를 가져와 계산하고, 결과를 다시 레지스터로 보냅니다.

### ② 제어장치 (Control Unit)

+ 명령어를 해독하고 시스템의 각 장치(기억, 연산, 입출력)에 제어 신호를 보냅니다.

+ 프로그램이 순서대로 실행되도록 관리하는 조율자 역할을 합니다.

### ③ 레지스터 (Register)

+ CPU 내부의 초고속 기억 장치입니다.

+ 연산에 필요한 데이터나 실행 중인 명령어 주소 등을 임시 저장합니다.

## 주요 특수 목적 레지스터

| 레지스터명 | 풀네임 | 역할 |
| :--- | :--- | :--- |
| **PC** | Program Counter | **다음에 실행할** 명령어의 주소를 저장 |
| **MAR** | Memory Address Register | 읽기/쓰기 연산을 수행할 **메모리 주소**를 저장 |
| **MBR** | Memory Buffer Register | 메모리에서 읽어온 데이터나 저장할 데이터를 **임시 저장** |
| **IR** | Instruction Register | **현재 실행 중인** 명령어를 저장 |
| **AC** | Accumulator | 연산 결과를 임시로 저장하는 **누산기** |

## CPU의 전체 동작 과정

1. **입력/읽기:** 주기억장치가 입력장치나 보조기억장치로부터 데이터를 읽어옵니다.

2. **처리:** CPU가 주기억장치의 명령어를 가져와 해독하고 처리한 뒤 결과를 다시 주기억장치에 보냅니다.

3. **저장/출력:** 주기억장치가 결과를 보조기억장치에 저장하거나 출력장치로 보냅니다.

4. **제어:** 제어장치가 위 과정이 순서대로 일어나도록 전체 장치를 제어합니다.

## 명령어 세트와 사이클

### 명령어 구조

+ **연산 코드(Op-Code):** 실행할 연산 (연산, 제어, 데이터 전달 등)

+ **피연산자(Operand):** 필요한 데이터 혹은 데이터가 저장된 위치(주소)

### 명령어 사이클 (Instruction Cycle)
CPU가 하나의 명령어를 처리하는 일련의 과정으로 **인출 → 실행 → 간접 → 인터럽트** 사이클로 구분됩니다.

## 인출 및 실행 사이클 (Micro-operations)

### 인출 사이클 (Fetch Cycle)

주기억장치에서 명령어를 가져와 IR에 저장하고, PC를 증가시키는 단계입니다.

```text
T0 : MAR <- PC            (PC의 주소를 MAR로 전달)
T1 : MBR <- M[MAR],       (메모리에서 명령어 인출)
     PC <- PC + 1         (다음 명령어를 위해 PC 증가)
T2 : IR <- MBR            (MBR의 명령어를 IR로 전달)
```

### 실행 사이클 (Execute Cycle) - ADD 명령어 기준

인출된 명령어를 실제로 연산하는 단계입니다. (PC 증가가 필요 없음)

```text
T0 : MAR <- IR(Addr)      (IR에 저장된 데이터 주소를 MAR로 전달)
T1 : MBR <- M[MAR]        (해당 주소의 데이터를 MBR로 가져옴)
T2 : AC <- AC + MBR       (누산기 값과 MBR 값을 더하여 AC에 저장)
```
