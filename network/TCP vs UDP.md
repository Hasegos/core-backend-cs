## TCP (Transmission Control Protocol)

TCP는 **데이터 통신 간에 안전하고 신뢰할 수 있는 방식으로 데이터를 주고받는 연결형 프로토콜**입니다.

### IP와 TCP의 비유

일반적으로 TCP와 IP는 함께 사용됩니다.

+ **IP (Internet Protocol):** 목적지까지 데이터를 나르는 **배달 기사** 역할.

+ **TCP:** 배달 앱의 **추적/관리 시스템** 역할 (배달 기사의 위치, 도착 여부, 물건 상태 확인 등).

### TCP의 주요 특징

+ **연결 지향 방식:** 데이터를 주고받기 전 송신지와 수신지 사이에 논리적인 연결 통로를 설정합니다. (**가상 회선 방식**)

+ **높은 신뢰성:** 패킷 분실 시 재전송을 요청하며, 데이터의 순서를 보장합니다.

+ **연결 및 해제 과정:** - **3-Way Handshake:** 세션을 수립하여 연결을 시작함.

   + **4-Way Handshake:** 세션을 종료하여 연결을 해제함.

+ **제어 기능:** 흐름 제어(Flow Control) 및 혼잡 제어(Congestion Control)를 통해 네트워크 상태에 맞춰 속도를 조절합니다.

+ **성능:** 제어 과정이 많아 UDP보다 속도가 상대적으로 느립니다.

+ **통신 방식:** 전이중(Full-Duplex), 점대점(Point to Point) 방식입니다.

### TCP 서버의 특징

+ 서버 소켓은 연결만을 담당하고, 실제 데이터 송수신은 반환된 클라이언트 소켓이 담당합니다.

+ 서버와 클라이언트는 1:1로 연결됩니다.

+ 스트림 전송 방식으로 데이터 크기가 무제한입니다.

+ 패킷 응답 확인 절차로 인해 실시간 스트리밍에는 다소 불리할 수 있습니다.

## UDP (User Datagram Protocol)

UDP는 데이터를 **데이터그램(Datagram) 단위로 처리하는 비연결형 프로토콜**입니다. 연결을 위한 논리적인 경로가 없으며, 각 패킷은 독립적으로 움직입니다.

<img width="500" src="https://github.com/user-attachments/assets/690c909d-20aa-48d5-a326-30c73b1478af" />

### UDP의 주요 특징

+ **비연결형 서비스:** 신호 절차(Handshake) 없이 데이터를 즉시 전송합니다.

+ **속도 중심:** 흐름 제어나 혼잡 제어를 하지 않아 TCP보다 빠르고 네트워크 부하가 적습니다.

+ **최소한의 오류 검출:** Checksum 필드를 통해 데이터 손상 여부만 확인합니다.

+ **낮은 신뢰성:** 데이터의 수신 여부나 전송 순서를 보장하지 않습니다.

+  **사용 사례:** 신뢰성보다 연속성이 중요한 **실시간 스트리밍(Streaming), 온라인 게임, 화상 통화** 등에 주로 사용됩니다.

### UDP 서버의 특징

+ 연결 개념이 없어 `connect()` 함수가 불필요하며, 서버/클라이언트 소켓 구분이 명확하지 않습니다.

+ 서버와 클라이언트는 1:1, 1:N, N:N 등 다양한 형태로 연결 가능합니다.

+ 데이터 전송 단위는 메시지(Datagram)이며, 최대 크기는 65,535바이트입니다.

## 핵심 개념: 흐름 제어 & 혼잡 제어

TCP가 신뢰성을 보장하기 위해 사용하는 핵심 메커니즘입니다.

+ **흐름 제어 (Flow Control):** 송신자와 수신자의 데이터 처리 속도 차이를 조절합니다. 수신자의 버퍼 오버플로우를 방지합니다.

+ **혼잡 제어 (Congestion Control):** 네트워크 내의 패킷 수가 과도하게 늘어나 정체가 발생하는 것을 방지합니다.

## TCP vs UDP 비교 요약

| 항목 | TCP | UDP |
| :--- | :--- | :--- |
| **연결 방식** | 연결 지향 (Virtual Circuit) | 비연결형 (Datagram) |
| **패킷 순서** | 보장함 | 보장하지 않음 |
| **신뢰성** | 높음 (재전송 존재) | 낮음 (재전송 없음) |
| **전송 속도** | 느림 | 빠름 |
| **데이터 경계** | 구분 안 함 (Stream) | 구분함 (Datagram) |
| **주요 사례** | 파일 전송(HTTP, FTP), 이메일 | 스트리밍, 게임, DNS |
