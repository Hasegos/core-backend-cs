## DNS란

<strong>DNS(Domain Name System)</strong>는 **도메인 네임과 IP 주소의 매핑 정보**를 관리하는 분산 데이터베이스 시스템입니다.<br>
숫자로 된 복잡한 IP 주소를 외울 필요 없이 문자열로 된 도메인을 통해 웹 사이트에 접속할 수 있게 해줍니다.

## DNS 동작 방식

기본적인 흐름은 다음과 같습니다.

1. **도메인 입력:** 사용자가 브라우저에 `www.google.com`을 입력합니다.

2. **IP 문의:** 브라우저는 먼저 PC의 캐시를 확인하고, 없으면 설정된 <strong>DNS 서버(네임 서버)</strong>에 해당 도메인의 IP를 물어봅니다.

3. **주소 반환:** DNS 서버는 데이터베이스에서 IP 주소를 찾아 브라우저에게 알려줍니다.

4. **접속:** 브라우저는 전달받은 IP 주소를 이용해 실제 서버에 접속합니다.

> 보통은 통신사(SKT, KT, LG 등)가 제공하는 기본 DNS를 사용하지만, 필요에 따라 구글(8.8.8.8)이나 클라우드플레어(1.1.1.1) DNS를 설정해 사용할 수도 있습니다.

## DNS 서버의 계층 구조

DNS는 전 세계의 모든 정보를 한 서버에 담을 수 없으므로 **계층적 구조**로 분산 관리됩니다.<br>
도메인의 마침표(`.`)를 기준으로 계층이 나뉩니다.

+ **Root Domain:** 모든 도메인의 뿌리. 보통 생략되지만 마지막에 붙는 `.`이 이에 해당합니다.

+ **TLD (Top Level Domain):** 최상위 도메인. `.com`, `.net`, `.kr` 등 국가나 목적을 나타냅니다.

+ **SLD (Second Level Domain):** TLD 하위 도메인. 서비스의 이름인 `google`, `naver` 등이 해당합니다.

+ **Subdomain:** 3단계 도메인. `www`, `blog`, `mail` 등 특정 서비스를 구분합니다.

## DNS 조회 과정 (Iterative Query)

실제로 DNS가 IP를 찾아오는 과정은 여러 네임 서버를 거치는 '재귀적/반복적' 탐색 과정입니다.

1. **Local DNS 캐시 확인:** PC나 브라우저, 공유기에 저장된 값이 있는지 확인합니다.

2. **Root 네임 서버:** "`.com` 관리하는 서버 주소 알려줘!" 요청.

3. **TLD(.com) 네임 서버:** "`.google.com` 관리하는 서버 주소 알려줘!" 요청.

4. **Authoritative(google.com) 네임 서버:** "`www.google.com`의 최종 IP 주소는 이거야!" 응답.

5. **결과 반환:** Local DNS 서버는 이 값을 캐싱하고 사용자 브라우저에 최종 IP를 전달합니다.

### 핵심 요약

+ **DNS:** 도메인 ↔ IP 주소 변환 시스템.

+ **계층 구조:** 루트(.) → 최상위(TLD) → 2단계(SLD) → 서브도메인 순으로 구성.

+ **분산 저장:** 상위 서버는 하위 서버의 위치(IP)를 알고 있어 순차적으로 찾아갈 수 있음.
