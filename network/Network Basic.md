## 네트워크란

네트워크란 <strong>노드(node)</strong>와 <strong>링크(link)</strong>가 서로 연결되어 리소스를 공유하는 집합을 의미합니다.

+ **노드:** 서버, 라우터, 스위치 등 네트워크 장치

+ **링크:** 유선(광케이블 등) 또는 무선(Wi-Fi, LTE 등) 연결 매체

## 처리량과 지연시간

웹 서비스의 성능을 평가하는 가장 핵심적인 지표 두 가지입니다.

### 처리량 (Throughput)

링크 내에서 **성공적으로 전달된 데이터의 양**을 말합니다.

+ **영향 요소:** 트래픽 양, 대역폭, 하드웨어 스펙.

+ **대역폭(Bandwidth):** 주어진 시간 동안 흐를 수 있는 데이터의 **최대 용량** (고속도로의 차선 수와 같음).

### 지연시간 (Latency)

요청이 처리되는 시간, 즉 데이터가 두 장치 사이를 
**왕복하는 데 걸린 시간**을 말합니다.

+ **영향 요소:** 유/무선 매체 타입, 패킷 크기, 라우터 처리 시간.

## 네트워크 토폴로지와 병목 현상

### 네트워크 토폴로지 (Topology)

노드와 링크가 배치된 연결 형태를 의미합니다.

| 종류 | 특징 | 장점 | 단점 |
| :--- | :--- | :--- | :--- |
| **트리** | 계층형 구조 | 확장이 쉬움 | 상위 노드 장애 시 하위 노드 영향 |
| **버스** | 단일 회선 공유 | 비용 저렴 | **스푸핑** 취약 |
| **스타** | 중앙 노드 집중형 | 관리 및 수리 용이 | 중앙 노드 장애 시 전체 마비 |
| **링형** | 고리 모양 연결 | 충돌 적음 | 회선 장애 시 전체 영향 |
| **메시** | 그물망 구조 | 높은 신뢰성 | 설치 및 운영 비용 고가 |

### 병목 현상 (Bottleneck)

전체 시스템의 성능이 특정 요소(가장 좁은 구간)에 의해 제한되는 현상입니다.

+ **해결 방법:** 경로 다중화(회선 추가), 대역폭 증설, 서버 스펙 업그레이드.

## 네트워크 분류

연결 범위(규모)에 따른 분류입니다. **(혼잡도: LAN < MAN < WAN)**

+ **LAN (Local Area Network):** 근거리 통신망 (집, 사무실).

+ **MAN (Metropolitan Area Network):** 도시 규모 네트워크.

+ **WAN (Wide Area Network):** 국가/대륙 규모 (인터넷).

## 네트워크 성능 분석 명령어
네트워크에 문제가 생겼을 때 원인을 파악하기 위해 사용합니다.

+ `ping`: 노드 연결 상태 확인 (ICMP 사용).

+ `netstat`: 서비스 상태 및 포트 연결 정보 확인.

+ `nslookup`: 도메인 이름으로 IP 주소 확인.

+ `tracert`: 목적지까지의 상세 경로 및 구간별 지연 확인.
