## 포트(Port)와 프로토콜

### 포트와 포트번호

IP 주소가 특정 컴퓨터를 찾는 주소라면, <strong>포트(Port)</strong>는 그 컴퓨터 안에서 실행되고 있는 특정 프로그램(서비스)을 찾는 문입니다.

+ **필요성:** 서버에서 HTTP, FTP, DNS 등 여러 서비스를 동시에 운영할 때, 클라이언트가 어떤 서비스에 접속할지 구분하기 위해 사용합니다.

+ **주요 포트 번호:**
    + HTTP: 80
    + HTTPS: 443
    + FTP: 20, 21
    + DNS: 53

### 프로토콜 (Protocol)
컴퓨터 간에 정보를 주고받기 위해 정의한 **통신 규약**입니다. "어떤 식으로 데이터를 주고받을지"에 대한 약속입니다.

## HTTP (HyperText Transfer Protocol)

HTTP는 인터넷에서 HTML 문서, 이미지, 동영상 등의 리소스를 주고받기 위한 기반 프로토콜입니다.

### 주요 특징

+ **클라이언트-서버 모델:** 클라이언트의 요청(Request)과 서버의 응답(Response)으로 동작합니다.

+ **무상태성 (Stateless):** 서버가 클라이언트의 이전 상태를 보존하지 않습니다. 각 요청은 독립적입니다.

+ **메서드:** GET(조회), POST(생성), PUT(수정), DELETE(삭제) 등.

+ **헤더(Header):** 데이터에 대한 메타데이터(Content-Type, Cache-Control 등)를 포함합니다.

### HTTP 메시지 구조

+ **요청 메시지:** 시작 행(메서드, URI, 버전), 헤더, 본문(Body)으로 구성됩니다.

+ **응답 메시지:** 시작 행(버전, 상태 코드, 상태 메시지), 헤더, 본문으로 구성됩니다.

## HTTPS와 SSL/TLS

**HTTPS**는 HTTP에 데이터 암호화 계층(SSL/TLS)을 추가하여 보안을 강화한 프로토콜입니다.

+ **보안성:** 모든 통신 내용을 암호화하여 중간자 공격(해킹)이나 데이터 변조를 방지합니다.

+ **인증:** 접속하려는 서버가 신뢰할 수 있는 서버임을 증명합니다.

+ **SEO:** 검색 엔진(Google 등)에서 보안이 적용된 HTTPS 사이트에 가산점을 부여합니다.

## 상태 유지 메커니즘 (쿠키 & 세션)

HTTP의 무상태성(Stateless)을 보완하여 사용자 정보를 유지하기 위해 사용합니다.

### 쿠키 (Cookie)

+ **정의:** 클라이언트(브라우저)에 저장되는 작은 텍스트 파일입니다.

+ **특징:** 서버를 거치지 않고 브라우저에서 바로 사용 가능하며, 보안에 다소 취약할 수 있습니다.

### 세션 (Session)

+ **정의:** 서버 측에서 클라이언트의 상태를 관리하는 메커니즘입니다.

+ **동작:** 서버는 고유한 **세션 ID**를 생성하여 클라이언트(쿠키)에게 전달하고, 실제 데이터는 서버 메모리나 DB에 저장합니다.

+ **장점:** 쿠키보다 보안성이 높지만, 서버 자원을 사용하므로 사용자가 많아지면 부하가 생길 수 있습니다.

> <strong>Tip: 캐시(Cache)</strong>는 상태 유지가 아닌, 자주 사용하는 데이터(이미지 등)를 미리 저장해두어 **속도를 높이는 용도**입니다.

## 자원 식별자 (URI vs URL)

웹상의 자원을 나타내는 방식에는 두 가지 개념이 있습니다.

### URI (Uniform Resource Identifier)

+ **통합 자원 식별자:** 인터넷상의 리소스 <strong>"자원 자체"</strong>를 식별하는 고유한 문자열입니다. 가장 포괄적인 개념입니다.

### URL (Uniform Resource Locator)

+ **자원 위치 지정자:** 네트워크상에서 리소스가 <strong>"어디에 있는지(위치)"<strong>를 나타내는 규약입니다.

+ **공식:** `URI = 식별자`, `URL = 식별자 + 위치`
