## 네트워크 기기의 처리 범위

네트워크 기기는 처리할 수 있는 계층에 따라 분류됩니다.<br>
핵심은 <strong>"상위 계층 기기는 하위 계층을 모두 이해하고 처리할 수 있지만, 역은 불가능하다"</strong>는 점입니다.

+ **애플리케이션 계층 (L7):** L7 스위치

+ **전송 계층 (L4):** L4 스위치 (로드밸런서)

+ **인터넷 계층 (L3):** 라우터, L3 스위치

+ **데이터 링크 계층 (L2):** L2 스위치, 브리지

+ **물리 계층 (L1):** NIC(LAN 카드), 리피터, AP

## 애플리케이션 계층을 처리하는 기기

### L7 스위치 (로드밸런서)

L7 스위치는 서버의 부하를 분산하는 **로드밸런서** 역할을 합니다.<br>
단순히 IP와 포트만 보는 것이 아니라, 실제 데이터(URL, 쿠키, 페이로드 등)를 분석하여 트래픽을 나눕니다.

+ **특징:** URL, 캐시, 쿠키 기반 트래픽 분산.

+ **헬스 체크(Health Check):** 서버에 주기적으로 요청을 보내 정상 동작 여부를 확인합니다.

+ **차이점:**

    + **L4 스위치:** IP와 포트 기반(NLB)으로 전송 계층에서 분산.

    + **L7 스위치:** 데이터 내용 기반(ALB)으로 응용 계층에서 분산.

## 인터넷 계층을 처리하는 기기

### 라우터 (Router)

여러 네트워크를 연결하고, 패킷이 목적지까지 가는 **최적의 경로**를 설정(Routing)하여 포워딩하는 장비입니다.

### L3 스위치

L2 스위치의 기능에 라우팅 기능을 추가한 장비입니다. 하드웨어 기반의 고속 라우팅을 수행합니다.

| 구분 | L2 스위치 | L3 스위치 (라우터 급) |
| :--- | :--- | :--- |
| **참조 테이블** | MAC 주소 테이블 | 라우팅 테이블 |
| **참조 PDU** | 이더넷 프레임 | IP 패킷 |
| **참조 주소** | MAC 주소 | IP 주소 |

## 데이터 링크 계층을 처리하는 기기

### L2 스위치

장치들의 **MAC 주소**를 관리하며 패킷을 해당 포트로 스위칭합니다.<br>
IP 주소를 기반으로 한 라우팅은 불가능합니다.

### 브리지 (Bridge)

두 개의 LAN을 연결하여 하나의 통신망을 구축할 때 사용합니다. 소프트웨어 기반으로 MAC 주소를 관리합니다.

## 물리 계층을 처리하는 기기

### NIC (Network Interface Card)

우리가 흔히 말하는 **LAN 카드**입니다.<br>
컴퓨터를 네트워크에 연결하며, 고유의 식별 번호인 **MAC 주소**를 가집니다.

### 리피터 (Repeater)

거리가 멀어져 감쇄된 전기 신호를 **증폭**하여 더 멀리 전달될 수 있게 합니다.

### AP (Access Point)

무선 신호를 유선 네트워크와 연결해 주는 장치입니다.<br>
공유기와 혼동하기 쉽지만, 순수 AP는 신호를 확장하고 전달하는 역할에 집중합니다.
