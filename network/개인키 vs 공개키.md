## 키(Key)란 무엇일까?

+ **정의:** 암호화(데이터 숨기기)와 복호화(원래대로 되돌리기) 과정에서 사용되는 <strong>'비밀 정보'</strong>이자 <strong>'암호문을 여는 열쇠'</strong>입니다.

+ **비유:** 우리 집 현관문을 열기 위한 '비밀번호'나 '열쇠'와 같습니다. 올바른 키 없이는 내부 데이터에 접근할 수 없습니다.

+ **수학적 관점:** 평문(Plaintext)과 키를 입력받아 암호문(Ciphertext)을 만드는 함수의 핵심 변수입니다.

## 대칭키 암호화 방식 (Symmetric Key)

송신자와 수신자가 **동일한 하나의 키**를 공유하여 사용하는 방식입니다.

+ **작동 원리:** 송신자가 '비밀키'로 암호화해서 보내면, 수신자도 '같은 비밀키'로 풀어냅니다.

+ **장점:** 구조가 간단하여 **연산 속도가 매우 빠르고** 대용량 데이터 처리에 유리합니다.

+ **단점:** 키를 전달하는 과정에서 유출될 위험이 크며, 관리해야 할 키의 개수가 사용자 수에 따라 급격히 늘어납니다.

## 비대칭키 암호화 방식 (Asymmetric Key)

<strong>공개키(Public Key)</strong>와 <strong>개인키(Private Key)</strong>라는 서로 다른 한 쌍의 키를 사용하는 방식입니다.

+ **작동 원리:**

    + **공개키:** 누구에게나 공개되는 키. 데이터를 **암호화**할 때 사용합니다.

    + **개인키:** 소유자만 가지는 비밀 키. 암호화된 데이터를 **복호화**하거나 **전자서명**을 할 때 사용합니다.

+ **비유:** 누구나 편지를 넣을 수 있는 '우편함 투입구(공개키)'와 주인만 열 수 있는 '우편함 열쇠(개인키)'의 관계와 같습니다.

+ **장점:** 키 분배가 안전하고, 데이터 기밀성뿐만 아니라 <strong>송신자 인증(전자서명)</strong>이 가능합니다.

+ **단점:** 대칭키 방식보다 계산이 복잡하여 **속도가 느립니다.**

## 대칭키 vs 비대칭키 비교

| 구분 | 대칭키 방식 (Symmetric) | 비대칭키 방식 (Asymmetric) |
| :--- | :--- | :--- |
| **키 구조** | 하나의 비밀키 (Shared Secret) | 공개키 + 개인키 한 쌍 (Key Pair) |
| **키 분배** | 키 전달 시 보안 채널 필요 (어려움) | 공개키는 자유롭게 배포 가능 (쉬움) |
| **처리 속도** | **매우 빠름** | 상대적으로 느림 |
| **용도** | 대량의 데이터 암호화 | 키 교환, 전자서명, SSL/TLS 초기 연결 |
| **사용 예시** | 파일 암호화, AES, VPN | HTTPS, SSH, 전자서명, RSA |

## 공개키와 개인키의 상세 역할

### 공개키 (Public Key)

+ **암호화:** 수신자의 공개키로 암호화하면, 그 메시지는 수신자의 개인키로만 열 수 있어 **기밀성**이 보장됩니다.

+ **서명 검증:** 송신자의 개인키로 만든 서명을 공개키로 풀어봄으로써 **송신자의 신원**을 확인합니다.

### 개인키 (Private Key)

+ **복호화:** 자신에게 온 암호화된 데이터를 원문으로 복구합니다.

+ **전자서명 생성:** 메시지를 자신의 개인키로 암호화(서명)하여 보내면, 수신자는 이를 통해 데이터의 **무결성**을 확인합니다.

+ **주의사항:** 유출 시 보안이 완전히 무너지므로 HSM(하드웨어 보안 모듈) 등 안전한 곳에 보관해야 합니다.

## 키 쌍(Key Pair) 생성 원리 (RSA 예시)

수학적으로 역계산이 어려운 문제를 이용합니다. (예: 거대 정수의 소인수분해)

1. 두 개의 큰 소수 $p, q$를 선택하여 $n = p \times q$를 구합니다.

2. 공개 지수 $e$를 정하고, $e \times d \equiv 1 \pmod{\phi(n)}$을 만족하는 비밀 지수 $d$ 를 구합니다.

3. **공개키:** $\langle e, n \rangle$ / **개인키:** $\langle d, n \rangle$ 쌍이 완성됩니다.

> **핵심 요약:** 실제 서비스(HTTPS 등)에서는 속도를 위해 **비대칭키로 대칭키를 안전하게 공유**한 뒤, 실제 데이터 전송은 **대칭키**로 수행하는 하이브리드 방식을 주로 사용합니다.
