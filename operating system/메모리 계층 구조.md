# 메모리 계층 구조 (Memory Hierarchy)

**메모리 계층 구조**는 용량, 접근 속도, 비용이라는 3가지 요소의 트레이드오프(Trade-off) 관계를 고려하여, CPU가 데이터에 최대한 빠르게 접근할 수 있도록 설계된 구조입니다.

## 계층 구조의 구성

메모리 계층은 CPU와 물리적/논리적 거리에 따라 단계별로 구성됩니다.

+ **상단 (CPU 인접):** 속도가 빠르고 비용이 비싸며 용량이 작음.

+ **하단 (CPU 원격):** 속도가 느리고 비용이 저렴하며 용량이 큼.

## 메모리 계층 비교표

| 단계 | 설명 | 주요 특징 |
| :--- | :--- | :--- |
| **레지스터 (Register)** | CPU 내부의 초고속 저장소 | 접근 속도 최고, 용량 최소, 휘발성 |
| **캐시 (Cache) L1~L3** | CPU와 RAM 사이의 완충 지대 | 자주 쓰는 데이터 저장, 지역성 원리 이용 |
| **주기억장치 (RAM)** | 메인 메모리 | 프로세스 실행 데이터 저장, 휘발성 |
| **보조기억장치 (SSD/HDD)** | 영구 저장 장치 | 비휘발성, 대용량 저장 가능, 속도 상대적 느림 |
| **외부 저장소 / 클라우드** | 네트워크 기반 저장소 | 물리적 제약 존재, 무한한 확장성 |

## 계층 구조가 필요한 이유

### ① 속도와 용량의 트레이드오프

+ 모든 저장 매체를 레지스터처럼 빠르게 만들면 비용이 감당 불가능합니다.

+ 반대로 모든 매체를 HDD처럼 크게 만들면 CPU가 노는 시간이 너무 많아집니다.

### ② 지역성 (Locality of Reference) 원리

+ 프로그램은 특정 시점에 특정 영역의 데이터만 집중적으로 참조하는 경향이 있습니다.

+ **시간 지역성:** 최근 접근한 데이터를 다시 접근할 가능성 높음.

+ **공간 지역성:** 인접한 주소의 데이터를 접근할 가능성 높음.

+ 이 원리 덕분에 작은 캐시만으로도 전체 시스템 성능을 획기적으로 끌어올릴 수 있습니다.

## 데이터 탐색 과정 (Hit & Miss)

CPU는 데이터를 찾을 때 상위 계층부터 하위 계층으로 순차적으로 탐색합니다.

> **탐색 순서:** 레지스터 → L1 → L2 → L3 → RAM → SSD/HDD → 네트워크

+ **캐시 히트(Cache Hit):** 필요한 데이터를 상위 단계(주로 캐시)에서 찾은 경우 → **성능 극대화**

+ **캐시 미스(Cache Miss):** 데이터를 찾지 못해 하위 단계로 내려가 가져오는 경우 → **지연(Latency) 발생**

## OS(운영체제)와의 연관성

컴퓨터 구조가 계층의 물리적 설계를 담당한다면, **운영체제(OS)는 이 계층을 소프트웨어적으로 관리하고 활용**합니다.

+ **가상 메모리:** 보조기억장치(HDD/SSD)를 마치 주기억장치(RAM)처럼 보이게 관리.

+ **페이지 교체 알고리즘:** RAM 용량이 부족할 때 어떤 데이터를 하위 계층으로 보낼지 결정 (LRU, LFU 등).

+ **TLB (Translation Lookaside Buffer):** 주소 변환 속도를 높이기 위한 특수 캐시 활용.

### 핵심 요약

> 메모리 계층 구조는 <strong>"CPU와 가까운 곳에는 빠르고 작은 저장장치, 먼 곳에는 크고 느린 저장장치"</strong>를 배치하여 **비용 효율성과 성능**을 동시에 잡기 위한 설계입니다.
