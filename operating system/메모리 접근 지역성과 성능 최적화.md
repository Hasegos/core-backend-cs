## 메모리 접근 지역성이란?

CPU와 메인 메모리 사이의 속도 차이를 극복하기 위해 캐시 메모리를 사용합니다. 이때 캐시에 어떤 데이터를 담을지 결정하는 기준이 바로 <strong>지역성(Locality)</strong>입니다.

## 시간 지역성 (Temporal Locality) 및 최적화

### 개념

+ 최근 접근했던 데이터나 명령어가 **가까운 미래에 다시 접근될 가능성이 높음**을 의미합니다.

+ **예시:** 루프(Loop) 내에서 반복적으로 사용되는 변수나 함수.

### 시간 지역성이 깨진 사례 (Broken Case)

`volatile` 키워드를 사용하거나 다른 스레드가 값을 계속 변경하면, CPU 캐시를 활용하지 못하고 매번 RAM에 접근해야 합니다.

```java
class Data {    
    public volatile int value = 10; // 메인 메모리 직접 접근 강제
}

// 매번 CPU 캐시 무효화 -> RAM 재접근 발생
for (int i = 0; i < 1_000_000_000; i++) {
    sum += data.value * data.value; 
}
```

### 최적화 방법 (Optimized Case)

자주 쓰이는 값을 <strong>지역 변수(Local Variable)</strong>에 담아 사용하면 CPU 레지스터 혹은 L1 캐시에서 연산이 이루어집니다.

```java
int local = data.value; // 딱 1번만 메모리 접근

for (int i = 0; i < 1_000_000_000; i++) {
    sum += local * local; // 이후 레지스터/캐시에서 초고속 연산
}
```

## 공간 지역성 (Spatial Locality) 및 최적화

### 개념

+ 접근된 메모리 주소 **근처(인접 주소)의 데이터가 곧 접근될 가능성이 높음**을 의미합니다.

+ **예시:** 배열의 순차적 접근.

### 공간 지역성이 깨진 사례 (Broken Case)

Java의 2차원 배열은 행(Row) 기준 연속 배치됩니다.<br>
만약 **열(Column) 기준으로 접근**하면 메모리 주소를 건너뛰게 되어 캐시 미스가 빈번해집니다.

```java
// 비연속 접근 (행을 점프하며 접근)
for (int j = 0; j < 5000; j++) {
    for (int i = 0; i < 5000; i++) {
        map[i][j]++; // [0][0] -> [1][0] -> [2][0] ... 캐시 효율 최악
    }
}
```

### 최적화 방법 (Optimized Case)

메모리 저장 순서와 동일하게 **행(Row) 단위로 순차 접근**합니다.

```java
// 연속 접근 (메모리 주소 순서대로)
for (int i = 0; i < 5000; i++) {
    for (int j = 0; j < 5000; j++) {
        map[i][j]++; // [0][0] -> [0][1] -> [0][2] ... 캐시 라인 활용 극대화
    }
}
```

## 요약 및 결론

| 지역성 종류 | 핵심 원리 | 최적화 전략 |
|---|---|---|
| 시간 지역성 | "방금 쓴 거 또 쓴다" | 자주 쓰는 값을 지역 변수나 레지스터에 캐싱 |
| 공간 지역성 | "옆에 있는 거 같이 쓴다" | 데이터가 저장된 순서(배열 등)대로 순차 접근 |

> **성능 최적화의 핵심:** 하드웨어의 캐시 메커니즘을 이해하고, <strong>캐시 히트(Cache Hit)<strong>율을 높일 수 있는 코드 패턴을 작성하는 것입니다.
