## 포인터(Pointer) 기초

포인터는 컴퓨터의 **메모리 주소**를 저장하는 변수입니다.

### 주요 연산자 및 형식

+ `%p`: 주소를 출력하는 형식 지정자 (16진수 출력)

+ `&`: **주소 연산자**. 변수의 주소를 알아냅니다. ("~의 주소가 어디야?")

+ `*`: **역참조 연산자**. 해당 주소로 이동하여 값을 가져옵니다. ("그 주소로 가줘")

> **예시:** `int *p = &n;`
> + `p`만 출력하면: `0x7ff...` (주소값)
>
> + `*p`를 출력하면: `50` (해당 주소에 저장된 실제 값)

## 배열과 포인터 (Array vs Pointer)

컴파일러는 배열 이름을 해당 배열의 **첫 번째 요소의 주소**로 해석합니다.

+ `a[1]` 은 내부적으로 `*(a + 1)` 과 동일하게 처리됩니다.

+ **차이점:** 배열은 선언 시 메모리가 자동 할당/해제되지만, 포인터를 이용한 동적 할당은 수동으로 관리해야 합니다.

+ **주의:** 문자열 비교 시 `==`를 쓰면 문자열 내용이 아닌 **주소값**을 비교하게 되므로 의도와 다를 수 있습니다.

## 문자열(String)의 본질

C에서 문자열은 사실 `char *` (첫 번째 문자의 주소)입니다.

+ `string s = "EMMA";` 에서 `s`는 첫 글자 'E'의 주소만 가리킵니다.

+ 컴퓨터는 주소를 1바이트씩 옮겨가며 읽다가 <strong>널 종단 문자(`\0`)</strong>를 만나면 문자열의 끝으로 인식합니다.

## 메모리 구조 (Memory Layout)
프로그램이 실행될 때 메모리는 용도에 따라 구역이 나뉩니다.

| 영역 | 설명 |
| :--- | :--- |
| **코드(Code)** | 컴파일된 기계어 명령어가 저장되는 곳 |
| **데이터(Data)** | 전역 변수, 정적 변수 등이 저장되는 곳 |
| **힙(Heap)** | `malloc` 등으로 **동적 할당**된 메모리가 위치하는 곳 (위로 쌓임) |
| **스택(Stack)** | 함수 호출 시 지역 변수, 매개변수가 저장되는 곳 (아래로 쌓임) |

### Swap 함수와 메모리

일반적인 값 복사 방식(`noswap`)은 스택 프레임이 사라지면 데이터도 사라지기 때문에 원본 값이 바뀌지 않습니다.<br>
<strong>주소값(Pointer)</strong>을 인자로 넘겨야 원본 메모리에 직접 접근하여 값을 교환할 수 있습니다.

## 메모리 할당과 해제 (malloc & free)

+ **malloc:** 힙 영역에서 메모리를 동적으로 할당받는 함수입니다. 에러 시 `NULL`을 반환합니다.

+ **free:** 할당받은 메모리를 명시적으로 해제해야 합니다. 하지 않으면 <strong>메모리 누수(Memory Leak)</strong>가 발생합니다.

+ **Valgrind:** 메모리 누수를 확인할 수 있는 유용한 디버깅 도구입니다.

## 파일 입출력 (File I/O)

+ `FILE *ptr = fopen("filename", "w");` 방식으로 파일을 엽니다. (`r`: 읽기, `w`: 쓰기, `a`: 추가)

+ `fprintf()`, `fread()`, `fwrite()` 등을 통해 데이터를 주고받습니다.

+ 사용이 끝난 파일은 반드시 `fclose(ptr);`로 닫아주어야 합니다.

+ **CSV 파일:** 쉼표(`,`)로 데이터를 구분하여 엑셀 등에서 열 수 있는 텍스트 파일 형식입니다.
