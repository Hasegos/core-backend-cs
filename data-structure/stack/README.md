## 개요 (Introduction)

스택은 데이터가 입력된 순서의 역순으로 나오는 **후입선출(LIFO)** 원칙을 따르는 자료구조입니다.<br>
한쪽 끝에서만 데이터의 삽입과 삭제가 일어나는 것이 특징이며, 주로 함수의 콜 스택(Call Stack), 뒤로 가기 기능, 괄호 검사 등에 활용됩니다.

[목차로 돌아가기](#목차)

## 주요 개념 (Key Concepts)

+ **LIFO (Last-In, First-Out):** 가장 마지막에 추가된 항목이 가장 먼저 제거되는 논리적 구조입니다.

+ **Top:** 스택의 가장 윗부분으로, 데이터의 삽입과 삭제가 발생하는 유일한 지점입니다.

+ **Push:** 스택의 맨 위에 새로운 데이터를 쌓는 작업입니다.

+ **Pop:** 스택의 맨 위에 있는 데이터를 꺼내고 제거하는 작업입니다.

[목차로 돌아가기](#목차)

## 스택의 주요 연산 (Key Operations)

스택의 핵심 동작은 다음과 같습니다.

+ **push(item):** 데이터를 스택의 최상단(Top)에 추가합니다.

+ **pop():** 스택의 최상단(Top) 데이터를 반환하며 제거합니다.

+ **peek():** 데이터를 제거하지 않고 최상단(Top)의 값만 확인합니다.

+ **isEmpty():** 스택이 비어 있는지 여부를 확인합니다.

## Java `Stack` 특징

프로젝트 구현 및 활용 시 고려한 Java `java.util.Stack` 클래스의 특징은 다음과 같습니다.

+ **Vector 상속:** Java의 `Stack`은 `Vector`를 상속받아 구현되어 있어 내부에 동기화(Synchronized)가 적용되어 있습니다. (Thread-Safe)

+ **Deqye 권장:** 멀티스레드 환경이 아닌 경우, Java 공식 문서에서는 `Stack` 대신 더 빠른 `ArrayDeque`를 사용할 것을 권장하고 있습니다.

+ **사용성:** 직관적인 메서드 명을 제공하여 후입선출 로직을 구현하기 매우 용이합니다.

+ **활용 사례:** [stack.java](./stack.java)

## 시간 복잡도 (Time Complexity)

| 연산 (Operation) | 평균 (Average) | 최악 (Worst) |
| :--- | :--- | :--- |
| **Push (삽입)** | $O(1)$ | $O(1)$ |
| **Pop (삭제)** | $O(1)$ | $O(1)$ |
| **Search (탐색)** | $O(n)$ | $O(n)$ |

> **Note:** 스택은 데이터의 삽입과 삭제가 항상 **Top**에서만 이루어지므로 $O(1)$의 매우 뛰어난 성능을 보입니다.<br>
하지만 특정 데이터를 찾는 탐색 작업은 모든 데이터를 확인해야 하므로 $O(n)$이 소요됩니다.
