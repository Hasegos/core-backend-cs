## Array vs Linked List

데이터를 연속적으로 저장하느냐, 흩어져서 연결되어 저장하느냐에 따라 성능 차이가 발생합니다.

### 1. Array (배열)

가장 기본적인 자료구조로, **논리적인 순서와 물리적인 순서가 일치**합니다.

+ **특징:** 메모리 상에 데이터가 연속적으로 붙어 있습니다.

+ **장점 (Access):** 인덱스(Index)를 알고 있다면 $O(1)$의 시간 복잡도로 즉시 접근할 수 있습니다. 이를 **Random Access**라고 합니다.

+ **단점 (Insertion/Deletion)**

    + 중간에 원소를 삭제하면 빈 공간을 메꾸기 위해 뒤의 원소들을 모두 한 칸씩 당겨줘야(**Shift**) 합니다.
    + 삽입 시에도 공간을 만들기 위해 뒤로 밀어줘야 합니다.
    + 이 과정에서 Worst Case 기준 $O(n)$ 의 비용이 발생합니다.

+ **구현 사례:** [ArrayList.java](./ArrayList.java)

### 2. Linked List (연결 리스트)

Array의 삽입/삭제 문제를 해결하기 위해 등장한 자료구조입니다.

+ **특징**: 각 원소(Node)는 **데이터**와 <strong>다음 노드의 주소(Pointer)</strong>를 가지고 있습니다. 물리적인 순서와 논리적인 저장 순서가 일치하지 않습니다.

+ **장점 (Insertion/Deletion):** 삭제나 삽입 시 다음 노드를 가리키는 주소값만 변경해 주면 되므로 작업 자체는 $O(1)$ 입니다.

+ **단점 (Search):** 특정 위치의 원소를 찾으려면 첫 번째 노드(Head)부터 순차적으로 탐색해야 합니다. 따라서 Search 과정에서 $O(n)$ 의 시간이 발생합니다.

+ **구현 사례:** [LinkedList.java](./LinkedList.java)

### 비교 요약

|기능|Array|Linked List|
|---|---|---|
| Random Access | $O(1)$ (매우 빠름)| $O(n)$ (순차 탐색 필요) |
|삽입/삭제 (맨 앞/뒤) | $O(n)$ (Shift 발생) | $O(1)$ |
|삽입/삭제 (중간) | $O(n)$ | $O(n)$ (탐색 비용 포함) |
| 메모리 할당 | 정적 (컴파일 타임) | 동적 (런타임) |

**실무적 선택 기준**: 데이터의 개수가 자주 바뀌지 않고 조회가 빈번하다면 Array<br>
데이터의 삽입과 삭제가 빈번하게 일어난다면 Linked List를 선택하는 것이 유리합니다.
