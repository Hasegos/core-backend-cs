## 개요 (Introduction)

트라이(Trie)는 'Retrieval(탐색)'에서 이름을 딴 자료구조로, 문자열의 각 문자를 노드로 구성하여 <strong>접두사(Prefix)</strong>를 공유하는 트리 구조입니다.<br>
사전 찾기, 자동 완성(Auto-complete), IP 라우팅 등 문자열 검색이 빈번한 시스템에서 최적의 성능을 발휘합니다.

## 주요 개념 (Key Concepts)

+ **Root Node (루트 노드):** 항상 비어 있는 상태이며, 모든 문자열의 시작점 역할을 합니다.

+ **Child Node (자식 노드):** 알파벳이나 숫자의 각 문자를 담고 있으며, 다음 문자로 연결되는 포인터를 가집니다.

+ **End of Word (단어 종결 표시):** 특정 노드에서 단어가 완성되었음을 나타내는 불리언(boolean) 플래그입니다.

## 트라이의 특징 (Features)

+ **빠른 문자열 검색:** 문자열의 길이만큼만 탐색하면 되므로, 저장된 단어의 수와 관계없이 매우 빠른 검색이 가능합니다.

+ **접두사 공유:** 공통된 접두사를 가진 단어들은 같은 노드를 공유하므로, '어근'을 기반으로 하는 검색에 최적화되어 있습니다.

+ **메모리 트레이드오프:** 탐색 속도는 빠르지만, 각 노드가 자식 노드 배열(예: 알파벳 26개)을 가져야 하므로 메모리 사용량이 상대적으로 많을 수 있습니다.

## 주요 연산 (Key Operations)

본 프로젝트에서는 트라이의 핵심 기능을 다음과 같이 구현하였습니다.

+ **Insert (삽입):** 문자열의 각 문자를 노드로 생성하며 트리를 내려가고, 마지막 노드에 종결 표시를 합니다.

+ **Search (탐색):** 입력된 문자열을 따라 노드를 이동하며, 마지막 노드의 종결 플래그가 `true`인지 확인합니다.

+ **Prefix Search (접두사 검색):** 특정 문자열로 시작하는 단어가 존재하는지 여부를 빠르게 판단합니다.

## 시간 복잡도 (Time Complexity)

문자열의 최대 길이를 $L$ 이라고 할 때:

| 연산 (Operation) | 복잡도 | 설명 |
| :--- | :--- | :--- |
| **Insert (삽입)** | $O(L)$ | 문자열의 길이만큼 노드를 생성 또는 이동합니다. |
| **Search (탐색)** | $O(L)$ | 문자열의 길이만큼 노드를 따라 내려가면 탐색이 종료됩니다. |

> **Note:** 해시 테이블은 평균 $O(1)$ 이지만 해시 충돌 시 $O(L \cdot n)$까지 성능이 저하될 수 있는 반면, 트라이는 저장된 단어의 개수와 상관없이 **항상 문자열 길이에 비례하는 일정한 속도**를 보장합니다.
