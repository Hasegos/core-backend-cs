스프링 부트는 스프링 프레임워크를 편리하게 사용할 수 있게 해주는 <strong>"자동화 도구"</strong>입니다.<br>

## 스프링 부트 핵심 기능 5가지

+ **내장 WAS:** Tomcat 같은 웹 서버를 내장하여 별도의 설치 없이 실행 가능.

+ **라이브러리 관리:** 빌드 구성 및 라이브러리 버전을 자동으로 관리.

+ **자동 구성 (Auto Configuration):** 스프링과 외부 라이브러리의 빈을 자동으로 등록.

+ **외부 설정:** 환경(Dev/Prod)에 따른 설정을 공통화하고 관리.

+ **프로덕션 준비:** 모니터링을 위한 메트릭, 상태 확인 기능 제공.

## War 배포 방식 vs 내장 서버 방식

### 1. 전통적인 외장 서버 (WAR)

+ **방식:** WAS(Tomcat) 별도 설치 → 코드를 WAR로 빌드 → WAS에 배포.

+ **단점:** 설정이 매우 복잡하며, 서버 환경과 코드 간의 버전 충돌 위험이 큼.

### 2. 스프링 부트 내장 서버 (JAR)

+ **방식:** 내장 톰캣 라이브러리를 포함하여 `main()` 메서드 실행으로 서버 구동.

+ **장점:** 배포가 단순하고 어디서나 동일한 환경에서 실행 가능.

## 내장 서버 설정 (원리 이해)

스프링 부트가 자동화해주는 과정을 코드로 이해하면 다음과 같습니다.

1. **톰켓 설정:** 내장 톰켓 인스턴스를 생성하고 포트를 연결.

2. **서블릿 등록:** `DispatcherServlet`을 생성하여 스프링 컨테이너와 연결.

3. **매핑:** 특정 경로(예: `/`)로 들어오는 요청을 디스패처 서블릿이 처리하도록 설정.

4. **실행:** `tomcat.start()`를 통해 웹 서버 구동.

> **핵심:** `main()` 메서드 하나로 웹 서버 실행부터 스프링 컨테이너 생성까지 한 번에 완료됩니다.

## 스프링 부트 실행 과정

```java
@SpringBootApplication  
public class BootApplication {  
    public static void main(String[] args) {  
       SpringApplication.run(BootApplication.class, args);  
    }    
}
```

`SpringApplication.run()` 한 줄이 실행되면서 발생하는 핵심 작업:

1. 스프링 컨테이너 생성: 빈(Bean) 등록 및 의존성 주입.

2. 내장 WAS(Tomcat) 생성: 서블릿 컨테이너 구동 및 디스패처 서블릿 연결.

## 실행 가능 JAR (Executable JAR)

과거의 Fat JAR는 모든 라이브러리를 압축 해제하여 클래스로 담았기 때문에 파일명 중복 문제가 발생했습니다.<br>
스프링 부트는 이를 "<strong>JAR 내부에 JAR를 포함하는 구조</strong>"로 해결했습니다.

### JAR 내부 구조

+ `META-INF/MANIFEST.MF`: 실행 클래스 정보 포함.

+ `org/springframework/boot/loader`: JAR 내의 JAR를 읽기 위한 `JarLauncher` 포함.

+ `BOOT-INF/classes`: 우리가 개발한 클래스 파일.

+ `BOOT-INF/lib`: 외부 라이브러리(JAR 파일들).

### 실행 흐름

1. `java -jar app.jar` 실행 시 `MANIFEST.MF` 인식.

2. `JarLauncher` 실행 (특별한 구조의 내부 JAR를 읽는 기술).

3. `JarLauncher`가 환경을 구성한 뒤 실제 `main()` 메서드(Start-Class) 호출.

## 총 정리

| 구분 | 과거 (WAR / Fat JAR)| 현재 (Spring Boot JAR) |
|----|----|----|
| WAS 관리 | 별도 설치 및 버전 관리 필요|애플리케이션 내장 (자동화)|
| 라이브러리 | 클래스 충돌 위험 (Fat JAR)| JAR 내 JAR 보관으로 충돌 방지|
| 배포 방식 |복잡한 배포 프로세스| JAR 파일 하나로 어디서든 실행|
| 생산성 |수동 설정의 어려움| 자동 구성 및 라이브러리 관리로 단축|
