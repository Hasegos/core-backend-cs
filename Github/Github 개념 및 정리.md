# Git 이란

Git은 <strong>분산형 버전 관리 시스템(Version Control System)</strong>으로, 프로젝트의 변경 사항을 스냅샷 기반으로 기록하며 빠른 수행 속도와 협업에 최적화되어 있습니다.

## Git의 필요성과 장점

### 왜 Git을 사용해야 하나요?

동시에 여러 명이 프로젝트를 작업할 때, 서로의 작업물에 영향을 주지 않고 수정본을 각각 업로드하여 합칠 수 있기 때문입니다.

### 주요 장점

+ 병렬 개발: 같은 파일을 여러 명이 동시에 작업하는 '병렬 개발'이 가능합니다.

+ 데이터 안전성: 중앙 저장소가 손실되어도 로컬 저장소를 통해 원상복구할 수 있습니다.

+ 오프라인 작업: 인터넷이 연결되지 않은 곳에서도 커밋 등 대부분의 개발 진행이 가능합니다.

## Git의 동작 원리 (Snapshot & Hash)

Git은 단순히 차이점을 기록하는 것이 아니라, 특정 시점의 상태를 통째로 저장하는 <strong>스냅샷(Snapshot)</strong> 방식을 사용합니다.

<img src="https://github.com/user-attachments/assets/7473def9-5f59-4374-848c-1984b3f21c85" />

### 해시(Hash)와 트리 구조

+ **해시값 인덱싱**: Git은 숫자가 아닌 고유한 해시값으로 버전을 식별하여 데이터 중복을 방지합니다.

+ **부모 노드 기록**: 각 노드(커밋)는 부모를 기록하여 전체적인 진행 순서와 계보를 유지합니다.

+ **분기(Branch)**: 기존 작업과 별개로 새로운 기능을 추가하고 싶을 때 특정 해시값을 참조하여 가지를 칠 수 있습니다.

## Git의 3가지 영역 (Workflow)

작업은 크게 세 가지 공간을 거쳐 진행됩니다.

1. **Working Directory**: 실제 파일 수정 및 저장 작업을 하는 개인의 디렉토리입니다.

2. **Staging Area**: 저장소에 커밋하기 전, 버전에 포함될 파일들을 준비시키는 장소입니다.

3. **Local/Remote Repository**: 최종적으로 확정된 스냅샷이 버전으로 저장되는 곳입니다.

## Git 버전 원리 이해하기

실제 작업 공간 에서 계산기를 만든다고 생각해보자

```js
let e = 10, f = 5;
let op = "+";

switch(op){
    case "+" :
        console.log(e + f);   
        break;
    default :
        break;
}
```
먼저 이상태를 1번이라 하고 더하기 기능을 만들었다.

```js
let e = 10, f = 5;
let op = "+";

switch(op){
    case "+" :
        console.log(e + f);
        break;   
    case "-" :
        console.log(e + f);   
        break;
    default :
        break;
}
```
위 상태는 2번이고 빼기 기능을 만들었다.

```js
let e = 10, f = 5;
let op = "+";

switch(op){
    case "+" :
        console.log(e + f);   
        break;
    case "-" :
        console.log(e + f); 
        break;
    case "*" :
        console.log(e * f);
        break;
        break;
    case "/" :
        console.log(e / f);
        break;  
    default :
        break;
}
```
위 상태는 3번이고 계산기 기능을 다 만들었다.

git에서는 각 작업물의 순서대로 번호순을 매겨줘서 버전 순서를 기억합니다.

하지만 git은 숫자로 인덱싱하지않고, 해시값으로 인덱싱을 해줍니다.

<img src="https://github.com/user-attachments/assets/3ecb083b-932c-4b0c-9e21-78cb34e491f9" />

해시값을줘서 중복은 안되는데, 해시값의 순서를 모른다. <br>
그래서 git에서는 트리처럼 부모를 각 노드마다 기록해주어 진행순서를 알게해준다.

만약 여기서 2번 뺴기 기능까지 만든것에서 다른기능을 추가하고싶다면 해당 해시값을
참조해서 가져오면된다.

```js
let e = 10, f = 5;
let op = "+";

switch(op){
    case "+" :
        console.log(e + f);
        break;   
    case "-" :
        console.log(e - f);   
        break;
    case "*" :
        console.log("해당 문구는 작업을 할 수 없습니다.")
        break;
    default :
        break;
}
```

위 처럼 기능을 추가하고 작업 순서를 저장을하게되면 기존에 만든 3번이랑 별개로 저장이 가능하다.

<img src="https://github.com/user-attachments/assets/08c94d92-8241-4eba-aad3-23fd0f54f47c" />

이런식으로 별개로 각각 작업해서 저장할 수 있는데, 이때 각각 어느 작업명인지 붙여준다. 

이를 ```분기점(Branch)```라고 한다.

## 📖 Git 핵심 용어 및 원리 정리

Git의 동작을 제대로 이해하기 위해서는 단순히 명령어만 아는 것이 아니라, 각 영역의 역할과 데이터가 저장되는 방식을 알아야 합니다.

### 1. 저장소 및 작업 공간

* **Working Tree (Working Directory)**: 사용자가 실제로 파일을 수정, 생성, 삭제하는 현재 시점의 작업 공간입니다.

* **Staging Area (Index)**: 커밋을 하기 전 '장바구니'에 담는 것과 같습니다. 수정한 파일 중 버전에 포함할 파일만 골라내는 영역입니다.
    * *Tip: 스테이지 내용은 `.git/index` 파일에 저장됩니다.*
* **Repository (저장소)**: 스테이지에서 대기하던 파일들을 하나의 버전(커밋)으로 만들어 영구히 저장하는 곳입니다.
    * **Local Repository**: 내 PC에 있는 개인 저장소입니다. (`git init`으로 생성)

    * **Remote Repository**: GitHub와 같은 서버에 있는 공유 저장소입니다.

### 2. 데이터 관리 개념

* **SnapShot (스냅샷)**: Git은 파일의 변경 차이점만 기록하는 것이 아니라, 특정 시점의 전체 상태를 통째로 캡처하여 저장합니다.<br> 이를 통해 어떤 시점으로든 완벽하게 되돌릴 수 있습니다.

* **Commit (커밋)**: 스냅샷을 찍어 저장소에 기록하는 행위입니다. 각 커밋은 고유한 해시값(ID)을 가집니다.

* **HEAD**: "나 지금 여기 있어"라고 말해주는 포인터입니다. 현재 작업 중인 브랜치의 최신 커밋을 가리킵니다.

### 3. 브랜치와 병합

* **Branch (브랜치)**: 원본을 유지한 채 새로운 기능을 시험해보고 싶을 때 만드는 '복사본 작업 줄기'입니다.

* **Merge (병합)**: 분리된 브랜치에서 작업이 완료되면 다시 메인 줄기로 합치는 작업입니다.

### 📌 Git 용어 한눈에 보기

| 용어 | 한 줄 정의 | 핵심 특징 |
| :--- | :--- | :--- |
| **Working Tree** | 실제 작업 공간 | 내 PC에서 눈에 보이는 폴더와 파일들 |
| **Staging Area** | 커밋 준비 영역 | 버전에 포함될 변경사항을 일시 저장 (`.git/index`) |
| **Repository** | 버전 저장소 | Local(내 PC)과 Remote(서버)로 구분됨 |
| **Snapshot** | 특정 시점의 상태 | 파일 전체 상태를 사진 찍듯 저장하는 방식 |
| **Commit** | 버전 기록 작업 | 변경 사항을 저장소에 영구히 저장하고 해시값 부여 |
| **Checkout** | 환경 전환 | 특정 브랜치나 과거 커밋으로 작업 위치를 이동 |
| **HEAD** | 현재 위치 포인터 | 현재 내가 체크아웃하여 작업 중인 지점 표시 |
| **Branch** | 분기점 (가지) | 독립적인 개발을 위해 나누어진 작업 흐름 |
| **Merge** | 병합 | 나누어진 브랜치들을 하나로 통합하는 과정 |
